# CustArch

Личный Bash-установщик Arch Linux и конфигурации системы. Этот файл — краткая карта проекта и единственный файл с инструкциями для работы с репозиторием.

## Как устроен запуск

```text
bootstrap.sh
  -> install.sh live
  -> arch-chroot /mnt /opt/custarch/install.sh chroot
  -> reboot
```

`install.sh` принимает фазу, загружает общие файлы из `lib/` и по имени запускает через `source` все `.sh`-стадии из соответствующего каталога `stages/`.

## Основные файлы

- `bootstrap.sh` — подготовка репозитория в Arch ISO и запуск установки.
- `install.sh` — общий runner фаз и стадий.
- `initial.conf` — начальные настройки пользователя и системы.
- `lib/common.sh` — сообщения, проверки, выбор и подтверждения.
- `lib/state.sh` — чтение и изменение состояния установки.
- `lib/packages.sh` — списки пакетов.
- `stages/` — небольшие последовательные шаги каждой фазы.
- `templates/` — конфигурации.

## Состояние установки

Состояние хранится в `/var/lib/custarch/state.env`. Каждый запуск фазы `live` создаёт его заново из `initial.conf`, а перед chroot копирует в установленную систему. Репозиторий копируется в `/opt/custarch`.

Не используй переменные окружения как единственный способ передать данные между фазами: отдельные фазы запускаются в разных процессах и системах.

## Текущая схема системы

- UEFI, `systemd-boot`, UKI через `mkinitcpio`.
- ESP смонтирован в `/boot`, UKI находится в `/boot/EFI/Linux/`.
- Корень на Btrfs.
- Подтома: `@`, `@home`, `@snapshots`, `@var_log`, `@pkg`.
- Swap-раздел не создаётся; используется `zram-generator`.
- Шифрование диска не нужно.

## Правила работы

- Сначала прочитай затрагиваемые скрипты и следуй уже существующей структуре.
- Делай небольшие понятные изменения; общую повторяемую логику выноси в `lib/`.
- Не форматируй и не выбирай диск неявно. Перед разрушительными действиями текстовое подтверждение.
- Не перезаписывай несвязанные пользовательские изменения.
- При изменении shell-файлов запускай `bash -n`; при наличии `shellcheck` также запускай его для изменённых файлов.
- Если запрос пользователя меняет описанную выше архитектуру, следуй запросу и одновременно обнови этот файл, чтобы он снова соответствовал проекту.
- не обращай внимания на SC1090,SC1091 в shellcheck. Можешь запускать shellcheck с игнорированем этих правил
